  • Recherche de properties

L'API permet de rechercher les properties correspondant à des critères géographiques, ou des critères de la property (surface, nombre de pièces...)

Critères de recherche

Pour effectuer une recheche il faut créer un objet de type CDVPropertySearch et lui définir les attributs à rechercher.

Il est possible de limiter une recheche sur des critères géographiques en utilisant des polygones, une distance (en kilomètres) autour d'un point, ou une bounding box


 CDVPropertySearch _propertiesSearch = new CDVPropertySearch();

 _propertiesSearch.setPageSize(10);
         _propertiesSearch.setService(CDVPropertyService.buy);
         _propertiesSearch.setParentTypes(new ArrayList<String>(Arrays.asList("house", "apartment")));
         _propertiesSearch.setNbRooms(new ArrayList<Double>(Arrays.asList(new Double(2), new Double(3))));

 CDVGeoPoint firstPoint = new CDVGeoPoint();
    firstPoint.setLat(52.36947753602144);
    firstPoint.setLng(-7.237292528124954);
    CDVGeoPoint secondPoint = new CDVGeoPoint();
    secondPoint.setLat(40.036761141167766);
    secondPoint.setLng(11.637219190625046);

    CDVGeoBoundingBox boundingBox = new CDVGeoBoundingBox();
    boundingBox.getPoints().add(firstPoint);
    boundingBox.getPoints().add(secondPoint);
    _propertiesSearch.setGeoBoundingBox(boundingBox);

Recherche pour obtenir une liste

Une fois l'objet CDVPropertySearch créé il est possible de recherche les properties correspondantes

       new CDVManager.PropertiesSearchTask(_propertiesSearch, new CDVManager.PropertySearchResponseHandler() {

            @Override
            public void result(CDVException e, ArrayList<CDVProperty> properties, int count) {
                if (e != null && e.getNativeException() == null) {
                    System.out.println("Search Properties Error : " + e.getErrorMessage());
                } else if (e != null && e.getNativeException() != null) {
                    e.getNativeException().printStackTrace();
                } else {
                    System.out.println(properties.size() + " properties returned with search on a total of " + count + " properties");


                }
            }
        }).execute();

Pour obtenir les résultats suivants (c-a-d la page suivante) il faut incrémenter la property cdvPage de l'objet CDVPropertySearch et relancer la recherche. La première page a l'index 1, la seconde page a l'index 2, etc-etc...


     propertiesSearch.cdvPage = 2

Recherche du nombre de properties

Cette recherche permet d'obtenir uniquement le nombre de properties correspondant à une recherche, et non détails sur les properties

 new CDVManager.CountPropertiesTask(_propertiesSearch, new CDVManager.CountPropertiesResponseHandler() {

                @Override
                public void result(CDVException e, int count) {
                    if (e != null && e.getNativeException() == null) {
                        System.out.println("Map Clusters Error : " + e.getErrorMessage());
                    } else if (e != null && e.getNativeException() != null) {
                        e.getNativeException().printStackTrace();
                    } else {
                        System.out.println(count + " properties");
                    }
                }
            }).execute();

ArrayList<Integer> _propertiesIds = new ArrayList<Integer>();
 _propertiesIds.add(42);
 _propertiesIds.add(686);

 new CDVManager.PropertiesByIdsTask(_propertiesIds, new CDVManager.PropertiesResponseHandler() {

                            @Override
                            public void result(CDVException e, ArrayList<CDVProperty> properties2) {

                                } else {
                                    System.out.println(properties2.size() + " properties returned by ids");

                                        System.out.println(_property.dictionary());
                                    }
                                }
                            }
                        }).execute();

Geo Precision (paramétrage)

The GeoHashgrid precision can either be defined in terms of the integer precision levels mentioned in the Elastic Search documentation. Values outside of [1,12] will be rejected. Alternatively, the precision level can be approximated from a distance measure like "1km", "10m" (cf. https://www.elastic.co/guide/en/elasticsearch/reference/current/search-aggregations-bucket-geohashgrid-aggregation.html)

propertiesSearch.cdvGeoPrecision = "2"

/* 2 types of possible values (String):
 * - GeoHashgrid code: '1', '2', ..., '12'
 * - Distances: '1m', '10km'
 */
